  Notes & mapping decisions (short)

Order visibility:

Customer & Vendor who placed the order: full order details (including contact/payment).

Goldsmith: only Order ID + product details + instructions (no personal info).

Admin: full access (assign, accept, change statuses).


Order ID generation: must be created server-side at checkout: DDMMYYSS (DD=day, MM=month, YY=year, SS=sequential index-of-day padded).

Vendor registration: public form stores submission in vendor_requests → Admin reviews → on Accept, Admin creates vendor account & system auto-generates Vendor ID.

Realtime rates: displayed in header; Admin can update via /admin/rates or connect to a rates API.

Images / uploads: all images are stored on Cloudinary; the app stores URLs in MongoDB.

Public Products view: must hide price & wastage for customers; vendor view (if needed) can include additional fields.

Notifications: push or in-app for Admin when goldsmith marks complete; for Goldsmith when assigned.
